## 📝 文档规范

每个插件文档包含：

1. **插件背景需求** - 为什么需要这个插件
2. **目标** - 插件要解决什么问题
3. **设计方案** - 插件的技术架构
4. **本地部署** - Win/Mac/Linux 部署指南
5. **效果展示** - 实际使用效果截图/录屏
6. **优缺点分析** - 插件的优势与不足
7. **平替对比** - 类似插件对比
8. **落地过程** - 实践验证的完整记录

## 🔧 如何贡献

1. 在 `docs/` 目录下创建插件文件夹
2. 按照上述规范编写 `README.md`
3. 在项目根目录 `README.md` 中添加链接
4. 提交 PR
