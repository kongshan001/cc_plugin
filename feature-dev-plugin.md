# Claude Code Feature Dev Plugin

> 官方功能开发工作流，7 阶段结构化方法

## 1. 背景需求

功能开发需要系统化的方法，从需求分析到实现测试，每个环节都很重要。Claude Code 官方推出 Feature Dev 插件，提供结构化的功能开发工作流。

## 2. 目标

通过 7 阶段结构化方法，引导开发者完成从需求分析到代码审查的完整流程。

## 3. 设计方案

### 7 阶段开发流程

```
1. 需求分析 → 2. 技术方案 → 3. 代码实现 
   ↓                                           ↓
7. 文档更新 ← 6. 测试验证 ← 5. 集成调试 ← 4. 单元开发
```

### 核心代理

| 代理 | 功能 | 阶段 |
|------|------|------|
| code-explorer | 代码库分析 | 需求分析 |
| code-architect | 架构设计 | 技术方案 |
| code-reviewer | 代码审查 | 全流程 |

### 阶段详情

**阶段 1: 需求分析**
- 理解功能需求
- 分析现有代码结构
- 确定边界和依赖

**阶段 2: 技术方案**
- 设计数据模型
- 确定接口设计
- 规划目录结构

**阶段 3: 单元开发**
- 核心逻辑实现
- 遵循项目规范
- 编写基础测试

**阶段 4: 集成调试**
- 模块集成
- 调试和问题排查
- 性能优化

**阶段 5: 测试验证**
- 单元测试
- 集成测试
- 端到端测试

**阶段 6: 代码审查**
- 自检代码质量
- 修复审查问题
- 确保合规性

**阶段 7: 文档更新**
- 更新 API 文档
- 编写使用说明
- 提交变更记录

## 5. 工作流示例

### 输入

```
功能: 用户密码重置
需求:
- 用户可以通过邮箱重置密码
- 发送重置链接到用户邮箱
- 链接有效期 24 小时
- 支持短信验证码备选
```

### 执行流程

```
[1] 需求分析
    - 分析现有用户系统
    - 确定重置流程
    
[2] 技术方案
    - 设计 PasswordReset 表
    - 定义 API 接口
    - 选择邮件服务
    
[3] 代码实现
    - 实现重置逻辑
    - 集成邮件服务
    - 添加缓存
    
[4] 集成调试
    - 集成测试
    - 调试邮件发送
    
[5] 测试验证
    - 单元测试覆盖
    - 集成测试通过
    
[6] 代码审查
    - 安全性检查
    - 性能优化
    
[7] 文档更新
    - API 文档
    - 更新 README
```

## 6. 优缺点分析

### 优点

✅ **结构化**: 7 阶段清晰流程
✅ **专业代理**: 代码分析、架构设计、代码审查
✅ **质量保证**: 每阶段有检查点
✅ **官方支持**: Anthropic 维护

### 缺点

⚠️ 较重流程，不适合小改动
⚠️ 需要完整需求描述
⚠️ 部分阶段可跳过

## 7. 平替对比

| 工作流 | 特点 | 适用场景 |
|--------|------|---------|
| Feature Dev | 7 阶段结构化 | 复杂功能 |
| RIPER Workflow | 5 阶段研究创新 | 创新功能 |
| TDD 流程 | 测试驱动 | 迭代开发 |
| Superpowers | 综合技能 | 全面开发 |

## 8. 落地过程

### 最佳实践

1. **清晰需求**: 提供完整功能描述
2. **阶段检查**: 每个阶段确认方向
3. **迭代改进**: 根据反馈调整
4. **完整测试**: 确保质量

### 配合使用

- **PR Review Toolkit**: 最终代码审查
- **TDD Guard**: 测试驱动开发
- **Security Guidance**: 安全检查
